# Hardware-Software Co-Design — RISC-V on FPGA

Custom RV32I processor implemented in VHDL on a Xilinx Zynq-7000 FPGA. Two-stage CPU with interrupts, memory-mapped peripherals, HDMI output, and a full autotest suite — runnable without Vivado via GHDL.

## Space Invaders on custom silicon

![Space Invaders simulation](frames/invaders.gif)

`C source → RISC-V .mem → GHDL simulation → VRAM log → PNG frames → GIF`

```bash
make compile_invaders_sim      # 1. compile for simulation
bash scripts/run_sim.sh        # 2. run GHDL (captures ~25 frames)
python3 scripts/render_frames.py  # 3. render frames/invaders.gif
```

## Prerequisites

| Tool | Notes |
|------|-------|
| Xilinx Vivado 2019.1 | source `settings64.sh` before use |
| `riscv32-unknown-elf-*` GCC | `-march=rv32i -mabi=ilp32` |
| GHDL | `brew install ghdl` — Vivado-free simulation |

Target FPGA: `xc7z010clg400-1`

## Build

`PROG=<name>` is always required — there is no default target.

```bash
make compile PROG=add                  # assemble to .mem
make compile PROG=invaders LIB=libfemto  # C + libfemto
make simulation PROG=add               # Vivado GUI sim
make simulation_cli PROG=add TIME=5000ns  # CLI sim (default 10000ns)
make synthesis PROG=compteur           # bitstream only
make fpga PROG=chenillard_rotation     # compile → synth → program FPGA
make autotest                          # RV32I tests (Vivado)
bash scripts/run_autotest.sh           # RV32I tests (GHDL)
make clean / realclean                 # clean artifacts
```

**Key Makefile variables:** `PROG` (required), `LIB` (`libfemto` for C), `TIME` (sim duration), `TOP` (top entity, default `PROC`)

## Architecture

```
PROC
├── CPU
│   ├── CPU_PC  — 37-state FSM (fetch / decode / execute)
│   ├── CPU_PO  — ALU, register file, memory ops
│   ├── CPU_CND — branch condition evaluator
│   └── CPU_CSR — CSR registers (mstatus, mepc, mcause, mtvec…)
├── RAM32       — 32 KB on-chip program memory
├── PROC_bus    — memory-mapped peripheral interconnect
└── Peripherals
    ├── IP_PLIC   — Platform-Level Interrupt Controller
    ├── IP_CLINT  — Core-Local Interruptor
    ├── IP_LED    — LEDs, switches, debug port (x31 writes)
    ├── IP_PIN    — push buttons
    ├── IP_Timer  — programmable timer
    └── HDMI      — AXI master + video encoder + TMDS
```

**Memory map**

| Address | Region |
|---------|--------|
| `0x0000_1000` | 32 KB on-chip RAM |
| `0x3000_0000` | memory-mapped peripherals |
| `0x8000_0000` | 256 MB DDR3 (AXI, HDMI only) |

All signals use strongly-typed VHDL enums defined in `vhd/PKG.vhd` — extend types there before adding new operations.

## Autotests (49/49)

```bash
bash scripts/run_autotest.sh        # all tests
bash scripts/run_autotest.sh add    # single test
```

Test format (`program/autotest/*.s`):

```asm
# TAG = add
    lui x1, 0x12345
    add x2, x1, x0
    # max_cycle 50
    # pout_start
    # 00000000
    # 12345000
    # pout_end
```

`pout_start/pout_end` values are compared against `x31` debug writes; `max_cycle` sets the simulation timeout.

## Adding an instruction

1. Add control signal types to `vhd/PKG.vhd`
2. Add FSM decode logic to `vhd/CPU_PC.vhd`
3. Add datapath operation to `vhd/CPU_PO.vhd`
4. Write a test in `program/autotest/<insn>.s`
